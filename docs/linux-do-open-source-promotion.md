## 项目事实素材

以下内容仅作为事实核对材料，不应原样粘贴为帖子正文。

- 项目名：html_ft_llm_any_style
- 在线预览：https://meomeo-dev.github.io/html_ft_llm_any_style/
- 仓库地址：https://github.com/meomeo-dev/html_ft_llm_any_style
- 技术形态：静态 HTML 工具，可直接本地打开。
- 页面入口：
  - `index.html`：工具主页，可切换对话风格调优器与
    代码风格调优器。
  - `ft_voice_style.html`：对话风格调优器。
  - `ft_code_style.html`：代码风格调优器。
- 功能事实：
  - 对话风格调优器包含 40+ 细粒度维度。
  - 代码风格调优器包含 50 个代码写法维度。
  - 支持预设一键应用。
  - 支持复制 YAML 或连续文本提示词。

## 人工项目介绍区

维护者在这里手写最终帖子正文。以下内容只用于提示写作方向，
不要原样发布。最终发布文本应由维护者用自己的经历、语气和判断
重新组织。

### 写作顺序建议

1. 先写项目动机：为什么需要一个 LLM 风格调优工具。
2. 再写使用场景：平时在哪些对话、代码生成或协作场景中用到。
3. 接着写项目能力：对话风格、代码风格、预设和复制输出。
4. 然后写项目边界：它只生成风格配置，不替代模型能力或业务判断。
5. 最后写反馈方向：希望社区反馈哪些维度、预设或使用问题。

### 可展开的问题

- 做这个项目之前，维护者遇到过哪些重复调 Prompt 的问题？
- 为什么把风格拆成细粒度维度，而不是只写一段自然语言提示词？
- 对话风格调优器更适合哪些用户：写作、客服、产品、研究还是日常助手？
- 代码风格调优器更适合哪些场景：代码审查、生成规范、团队协作还是
  个人偏好记录？
- 一键复制 YAML 和连续文本提示词，分别适合接入哪些工作流？
- 当前静态 HTML 形态有什么好处：无需登录、本地可用、容易审查、
  容易 fork。
- 当前版本还缺什么：更多预设、导入导出、主题适配、更多截图、
  示例配置或英文界面。

### 可引用事实

- 项目是静态 HTML 工具，无需后端服务。
- 项目当前包含对话风格调优器和代码风格调优器。
- 对话风格调优器包含 40+ 细粒度维度。
- 代码风格调优器包含 50 个代码写法维度。
- 工具支持预设一键应用。
- 工具支持复制 YAML 或连续文本提示词。
- 项目提供 GitHub Pages 在线预览，也可以克隆后本地打开。

- 可表达的判断：自然语言 Prompt 容易混在一起，拆成维度后更容易复用。
